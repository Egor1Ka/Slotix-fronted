# Фильтрация услуг и сотрудников на публичной странице организации

## Цель

Исправить логику публичной страницы бронирования организации (`/org/[orgId]`): добавить двустороннюю фильтрацию между услугами и сотрудниками, убрать общий вид, сделать выбор исполнителя обязательным.

## Проблемы сейчас

1. При выборе услуги в сайдбаре — табы сотрудников не фильтруются. Показываются все, даже те кто не может эту услугу выполнять.
2. При выборе сотрудника — услуги в сайдбаре не фильтруются. Показываются все услуги организации.
3. На общем виде (ни один сотрудник не выбран) — рисуются серые блоки заблокированного времени, хотя у конкретных сотрудников это время свободно.
4. Нет обязательного выбора исполнителя — можно попытаться забронировать без конкретного сотрудника.

## Решение

### Бэкенд

**Новый эндпоинт:** `GET /event-types/:id/staff`

Возвращает список сотрудников организации, которые могут выполнять данную услугу (EventType). Логика определяется полем `staffPolicy` в EventType:

- `"any"` — все активные сотрудники организации (через `getActiveMembersByOrg`)
- `"by_position"` — сотрудники, чей `positionId` входит в `assignedPositions` EventType
- `"specific"` — сотрудники, чей `userId` входит в `assignedStaff` EventType

Ответ: массив `OrgStaffMember[]` (тот же формат что и `GET /org/:id/staff`).

**Файлы для изменения:**
- `src/services/eventTypeServices.js` — новая функция `getStaffForEventType(eventTypeId)`
- `src/controllers/eventTypeController.js` — новый хендлер `handleGetStaffForEventType`
- `src/routes/subroutes/eventTypeRoutes.js` — новый роут `GET /:id/staff`

### Фронтенд — публичная страница `/org/[orgId]`

#### Двусторонняя фильтрация

**Состояние:**
- `selectedStaffId: string | null` — выбранный сотрудник (первый по умолчанию)
- `selectedEventTypeId: string | null` — выбранная услуга (null = не выбрана)
- `filteredStaff: OrgStaffMember[]` — отфильтрованные сотрудники
- `filteredEventTypes: EventType[]` — отфильтрованные услуги

**Логика фильтрации:**

1. **Ничего не выбрано** — слева все услуги организации, вверху табы всех сотрудников. Первый сотрудник выбран по умолчанию. Календарь показывает расписание первого сотрудника.

2. **Выбрал сотрудника** — услуги в сайдбаре фильтруются через `GET /event-types?staffId=X`. Остаются только услуги, которые этот сотрудник может выполнять.

3. **Выбрал услугу** — табы сотрудников фильтруются через `GET /event-types/:id/staff`. Остаются только сотрудники, которые могут выполнять эту услугу. Если текущий выбранный сотрудник не входит в отфильтрованный список — автоматически выбирается первый из отфильтрованных.

4. **Выбраны оба** — календарь показывает слоты для конкретной пары (staffId + eventTypeId). Услуги отфильтрованы по сотруднику, сотрудники отфильтрованы по услуге.

#### Общий вид убран

На публичной странице нет состояния "никто не выбран". При загрузке всегда активен первый сотрудник из списка.

#### Обязательный выбор исполнителя

Нельзя забронировать без выбранного сотрудника. Опции "Будь-хто" (любой) нет.

### API-клиент (фронтенд)

Новый метод в `booking-api-client.ts`:
```typescript
eventTypeApi.getStaff(eventTypeId: string): Promise<OrgStaffMember[]>
```

Вызывает `GET /api/event-types/:id/staff`.

### Что НЕ меняется

- **Админские страницы** (`/manage/[orgId]`) — без изменений. Админ видит всех сотрудников и все услуги.
- **Личное расписание** (`/schedule`) — без изменений.
- **Публичная страница соло-мастера** (`/book/[staffSlug]`) — без изменений. Там один мастер, фильтрация не нужна.
